# Task Bridge SaaS (TaskFlow) — システム仕様・利用マニュアル

## 1. システム概要

Task Bridge SaaS は、社内の各種申請・承認ワークフローを一元管理し、業務の透明性と処理効率を向上させるための SaaS プラットフォームです。  
部署間の依頼（名刺作成・PC 手配・アカウント権限付与など）をデジタル化し、動的な承認ルート・優先度管理・監査ログ・下書き保存など、実務に即した機能を備えています。

---

## 2. 対象ユーザーとロール（権限）設計

ログインするユーザーの役職・部署に応じて、インターフェースと権限が自動的に切り替わります。

| ロール | 対象者 | 主な権限 | 主な画面 |
|--------|--------|---------|---------|
| 一般社員 | Member / Team Leader など | 申請作成・自分の進捗確認 | 新規申請、トラッカー |
| 承認者 | Team Leader / General Manager | 申請の承認・差し戻し・コメント | 依頼受付・管理（Inbox） |
| 管理者 | Admin ロール | 全申請の監視・組織管理・設定変更 | 管理者設定・全社監査ログ |

---

## 3. 主要機能

### 3.1 新規申請・依頼作成

- **動的フォーム**: カテゴリー（総務・人事・IT など）に応じた入力項目が動的に変化
- **優先度設定**: 低 / 通常 / 急ぎ の3段階を選択可能。カード・詳細画面にバッジ表示
- **SLA 自動計算**: カテゴリーごとの標準処理日数から期限を自動設定
- **承認ルートの自動生成**: 申請者の上長ヒエラルキーに基づいてルートを自動構築。手動での追加・削除も可能
- **CC（共有先）機能**: 承認ルートとは別に共有先メンバーを追加（アバターバッジ形式）
- **ファイル添付**: ドラッグ＆ドロップまたはクリックでファイルを添付可能
- **下書き自動保存**: 入力内容を 500ms デバウンスで `localStorage` に保存（最大5件）。次回アクセス時に選択復元バナーを表示。送信後は自動削除

### 3.2 依頼受付・管理（Inbox）

- **ロール別表示**: 一般ユーザーは自分が承認者のタスクのみ表示。管理者は全件表示
- **タブフィルター**: すべて / 未着手 / 対応中 / 期限超過 / 完了済み
- **キーワード検索**: タイトル・カテゴリー名でリアルタイム絞り込み
- **期間フィルター**: 今日 / 今週 / 今月 / 全期間 のドロップダウンで絞り込み
- **ソート切り替え**: 新しい順 / 古い順 / 件名順
- **承認・差し戻し**: 自分の順番が来た申請に対してワンクリックで処理
- **コメント機能**: 申請に対してインアプリメッセージ感覚でコメント投稿
- **監査ログ（タイムライン）**: 申請ごとの全アクション履歴を時系列表示

### 3.3 自分の依頼状況（トラッカー）

- **進捗バー・ステッパー**: 承認ルートの現在ステップをカード・詳細モーダルで視覚表示
- **キーワード検索・ステータスフィルター**: 申請を素早く特定
- **申請キャンセル**: 未処理の申請（全承認者が未処理）を取り消し可能。監査ログに記録
- **差し戻し後の再申請**: 差し戻された申請を修正して同一 ID で再送信
- **再利用機能**: 完了済み申請の内容を引き継いで新規申請を作成

### 3.4 管理者設定（Admin）

- **基本設定**: テナント名を編集して保存（`localStorage` に永続化）
- **組織・メンバー管理**:
  - メンバー一覧の表示（ステータス・役職・所属ユニット）
  - 新規メンバー追加フォーム
  - CSV 一括インポート（`importBulkData()` 経由）
  - 行ごとの MoreHorizontal メニューから人事・組織変更モーダルを呼び出し
  - 人事・組織変更（異動 / 退職 / 休職 / 昇格）の登録と即時適用。対象タスクの担当者を自動引き継ぎ
- **申請カテゴリーマスタ**: カテゴリーの追加・編集・削除（SLA・大分類の設定）
- **全社監査ログ**: ユーザー名・アクション種別でのキーワード検索。該当タスクへのワンクリックジャンプ
- **セキュリティ・権限**: 将来実装予定（現在は RBAC ポリシーをデフォルト適用）

### 3.5 組織図（Directory）

- **インタラクティブ組織図**: ツリー形式で会社の部署構成・メンバーを展開表示
- **ユーザー検索**: 氏名・役職・メールアドレスで検索（300ms デバウンス）。結果クリックでサイドパネルに詳細表示
- **プロフィール詳細モーダル**: メール・役職・ステータス・入社日などを一覧表示

### 3.6 ナビゲーション全体

- **ヘッダー**: 通知パネル（Bell）、ヘルプモーダル（HelpCircle）、プロフィール設定・環境設定（ドロップダウン）
- **サイドバー**: 社内規定・規約モーダル、サポート・ヘルプモーダル

## 4. データモデル概要

| モデル | 主なフィールド |
|--------|--------------|
| `User` | id, name, email, role, position, orgUnitId, managerId, status, avatar, joinedAt |
| `Task` | id, title, description, categoryId, statusId, status, priority, approvalRoute, ccRoute, currentApproverId, customData |
| `ApprovalStep` | userId, userName, position, avatar, status, comment, processedAt |
| `Category` | id, name, parentId, targetDepartmentId, slaDays, customFields |
| `AuditLog` | id, taskId, userId, action, comment, timestamp |
| `UserChangeEvent` | id, targetId, eventType, scheduledAt, changes, status |

## 5. 技術スタック

| レイヤー | 技術 |
|---------|------|
| Framework | Next.js 16 (App Router) |
| UI Library | React 19 |
| Styling | Tailwind CSS |
| Icons | Lucide React |
| 型システム | TypeScript（strict モード） |
| データ層 | In-memory Mock Provider（`src/lib/repository/mock-provider.ts`） |
| 状態管理 | React useState / useCallback / useMemo / useRef |
| 永続化（クライアント） | localStorage（下書き・設定保存） |

> **注意**: 現在はインメモリ Mock を使用しており、ページリロードでデータがリセットされます。  
> `src/lib/repository/factory.ts` の `setDataProvider()` を呼び出すことで、Supabase 等の実 DB プロバイダーへ切り替え可能です。

## 7. 利用マニュアル（ユーザー向けフロー）

### 7.1 申請書の出し方

1. サイドバーの「新規申請・相談」をクリック
2. カテゴリー（例: 総務 → 名刺作成）を選択
3. 優先度・タイトル・詳細内容・追加項目を入力
4. 必要に応じて CC（共有先）を追加
5. 「申請を送信する」をクリック

> 入力中の内容は自動保存されます。途中でページを離れても、次回アクセス時に「下書き一覧」から復元できます。

### 7.2 承認の仕方（マネージャー・リーダー向け）

1. サイドバーの「依頼受付・管理」を開く
2. 「未着手」タブにある自分宛の申請を選択
3. 内容を確認し「承認・受理する」または「差し戻し」をクリック

### 7.3 差し戻し後の再申請

1. 「自分の依頼状況（トラッカー）」で差し戻しになった申請を開く
2. 「修正して再申請する」をクリック
3. 内容を修正して「修正内容を再申請する」をクリック

### 7.4 申請のキャンセル

1. 「自分の依頼状況（トラッカー）」で対象の申請を開く
2. 「申請をキャンセル」ボタンをクリック（承認者が誰も処理していない場合のみ表示）
