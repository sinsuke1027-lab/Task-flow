# Task Bridge SaaS (TaskFlow) - システム仕様・利用マニュアル

## 1. システム概要
Task Bridge SaaS は、社内の各種申請・承認ワークフローを一元管理し、業務の透明性と処理効率を向上させるためのSaaSプラットフォームです。
部署間の依頼（例：名刺作成、PC手配、アカウント権限付与など）をデジタル化し、動的な承認ルート、監査ログ、再利用機能などを備えています。

## 2. 対象ユーザーとロール（権限）設計
システムはログインするユーザーの役職や部署に応じて、自動的に適切なインターフェースと権限を提供します。

### 2.1 ユーザーロール
1. **一般社員 (General User)**
   - **権限**: 自身の申請の作成、自身の申請状況の追跡（トラッカー）、承認完了後の確認。
   - **主な画面**: 新規申請、トラッカー（自分の依頼状況）。
2. **マネージャー / 承認者 (Team Leader / General Manager)**
   - **権限**: 一般社員の権限に加え、自身の部署宛ての申請の審査（承認・却下・差し戻し）。
   - **主な画面**: レビュー（あなたの確認待ち）、ダッシュボード（部署内の状況）。
3. **システム管理者 (Admin)**
   - **権限**: システム全体の俯瞰、全申請の監視、全社監査ログへのアクセス、システム設定。
   - **主な画面**: 管理ダッシュボード、全社監査ログ（Audit Logs）。

## 3. 主要機能 (Feature Specifications)

### 3.1 新規申請・依頼作成 (Request Form)
- **動的フォーム**: 選択された申請カテゴリー（総務、人事、ITなど）に応じて、必要な入力項目が動的に変化します。
- **SLA（期限）の自動計算**: カテゴリーごとに設定された標準処理日数（SLA）に基づいて、自動的に申請の期限が設定されます。
- **承認ルートの自動生成**: 申請内容と部署に基づいて、必要な承認ルート（ステップ）が自動的に生成されます。
- **CC（共有先）機能**: 承認ルートとは別に、進行状況を共有したいメンバー（閲覧・コメント可能権限）をアバター付きのバッジ形式で追加できます。

### 3.2 インボックス / 案件管理 (Inbox & Tracker)
- **ステータス管理**: 申請は「未着手 (todo)」「対応中 (in_progress)」「完了 (completed)」「期限超過 (overdue)」のステータスで管理されます。
- **フィルタリング機能**: ロールに応じて適切なタブ（すべて、未着手、対応中等）が提供され、瞬時に目的の案件にアクセス可能です。
- **詳細ビュー**:
  - **タイムライン (Audit Logs)**: 各申請におけるすべてのアクション（送信、承認、コメント等）とタイムスタンプを履歴として表示します。
  - **インアプリメッセージング**: 申請に関する質問や連絡事項をチャット感覚でコメント投稿できます（CCユーザーも参加可能）。
  - **再利用機能 (Reuse)**: 完了済みの案件をベースに、入力内容やCC設定を引き継いだ新規の申請をワンクリックで作成できます。

### 3.3 全社管理機能 (Admin Features)
- **全社監査ログ**: 管理者限定の画面で、システム内で発生した全ユーザー、全申請のアクション履歴を一覧表示・検索が可能です。
- **KPIダッシュボード**: 申請の処理件数、過負荷状態の部署、ボトルネックなどを視覚的なチャートで分析できます。

### 3.4 組織図とユーザーディレクトリ (Directory)
- **インタラクティブ組織図**: 会社の部署構成やメンバーをツリー形式等で俯瞰できます。
- **ユーザー検索**: スキルや部署から特定の人物を探し出し、そのままダイレクトに依頼を作成できます。

## 4. データモデル概要
本システムは現在フロントエンド（モック）で動作しており、主に以下のデータ構造を持っています。

- **User**: ユーザー情報（ID, 名前, 部署, ロール, アバターなど）
- **Task**: 申請データ（ID, タイトル, 説明, カテゴリー, ステータス, 期限, 承認ルート (`approvalRoute`), 共有先 (`ccRoute`), カスタムデータなど）
- **ApprovalStep**: 承認・共有のパス情報（ユーザーID, ステータスなど）
- **AuditLog**: 履歴データ（操作ID, タスクID, アクション内容, ユーザー, タイムスタンプ）

## 5. 技術スタック
- **Frontend Framework**: Next.js (App Router), React
- **Styling**: Tailwind CSS
- **Icons**: Lucide React
- **Mock Data**: in-memory Provider (`src/lib/repository/mock-provider.ts`)

## 6. 利用マニュアル（ユーザー向けフロー）

### 6.1 申請書の出し方
1. サイドメニューから「新規申請・相談」をクリックします。
2. 依頼したいカテゴリー（例：「総務部」→「名刺作成」）を選択します。
3. タイトル、詳細内容、およびカテゴリー特有の追加項目（必要枚数など）を入力します。
4. （任意）情報を共有したいメンバーがいれば、「CC（共有先）を追加」から検索して追加します。
5. 「申請を送信する」をクリックして完了です。

### 6.2 承認の仕方（管理者・リーダー向け）
1. サイドメニューから「インボックス」を開きます。
2. 「未着手」タブにある案件が、あなたの対応を待っている申請です。
3. 申請を選択し、内容から問題がなければ右下の「承認・受理する」をクリックします。修正が必要な場合は「差し戻し」をクリックします。

### 6.3 過去の申請を再利用する
1. 「インボックス」または「トラッカー」で、すでに「完了済み」になっている申請を開きます。
2. 画面右下の「再利用して申請」をクリックします。
3. 前回の入力内容とCC設定が自動で引き継がれた新規フォームが開くので、必要な部分だけ修正して送信します（タイトルの先頭に接頭辞は付きません）。
